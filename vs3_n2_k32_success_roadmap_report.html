1. 总体判断

Confirmed

七类干预全部没有移动失败曲线

dt÷20（fine5/fine20/staged/full20）、rcond 10⁻⁴→10⁻⁶、A-seed、COMdiag+去 gauge+宽窗——所有 gauss 变体在相同相位以三位小数一致的方式偏离 grid。误差在 \(s\approx0.13\)（\(\phi\approx0.42\pi\)）已经形成，早于一切 fine window。

Key positive

free + rcond 10⁻⁵ 已几乎通过第一次跃迁

frozen-A 160 维流形在 rcond=10⁻⁵ 下保持 \(|\delta\Delta P|\le 0.006\) 直到 \(\phi=0.50\pi\)。Gate 1 对 ECG 是可通过的——失败不是方法本身不行，而是 gauss 分支的求解配置不行。

Cheapest missing test

E1（gauss·A frozen·rcond 10⁻⁵）从未跑过

相互作用对极化的物理影响只有 \(10^{-3}\) 量级；free 的成功配置直接加上 Gaussian 势（不动 A）是最短的成功路径候选，代价只有几小时。

Engineering blocker

没有 checkpoint/restart，长 run 必然撞墙

COMdiag gauss 计划 138.5 h > 99 h walltime 上限；全周期任何变体都要 1–6 天。工程缺口（restart、宽度守护、basis 修剪）本身就是科学瓶颈的一半。

本报告的中心论点：第一次跃迁处，TDVP 线性解把 25%–56% 的梯度范数整体截断（discarded_rhs_fraction 峰值；N=1 成功 run 一生的峰值只有 10.5%）。全局 rcond 阈值作用在未做尺度平衡的 metric 上，把"低振幅 pad 的形状/位置/相位自由度"成批冻结——其中既有应当停车的空 reserve（这是 basis 的设计意图，见 §4-H1），也有不该被截的已占据与正在苏醒的方向（有害部分）。成功 = 先用最小流形绕过这个陷阱（R1/R2），同时用占据门控白化修正已占据方向的截断、并显式保留 reserve 的停车性质（R3 修订版），再修长程 width/rank 崩塌（R7），最后才恢复 σ 扫描（R8）。

3. 已被数据证伪的假设

3.1 "时间步长不够小"（失败报告 E3）——已证伪

full20 / full20_repeat 把 全周期 dt 固定为 \(5\times10^{-4}\)（比 base 粗步小 20 倍，从 t=0 起、无窗口混杂）。σ=0.75 已走到 \(0.50\pi\)、σ=1 走到 \(0.496\pi\)： 与 base（dt=0.01）的误差曲线在所有相位点重合（0.01-越界点 0.425π vs 0.427π）。RK4 在 dt≤0.01 已经收敛——只是收敛到同一条错误的流形轨迹上。继续为 dt 花机时没有意义。

3.2 "rcond 选错了"——单独调阈值已证伪

gauss 在 rcond 10⁻⁴/10⁻⁵/10⁻⁶ 下曲线一致。原因见 §4-H1：被截掉的方向的奇异值比 \(\mathrm{rcond}\cdot\sigma_{\max}\) 低若干个数量级，阈值在可扫范围内挪动不改变保留集合的结构。 注意对照组：free 对 rcond 敏感且 10⁻⁵ 明显更好——说明"保留更多方向"本身有价值，只是 gauss 的 256 维 metric 里被截方向的结构不对。

3.3 "预埋 A-seed 能救"——设计无效 + 数据无变化

失败报告 §4 已给出五条设计缺陷（seed 落在近零 reserve 上、漏 u 补偿、被 rcond 截断等）；本轮数据确认两个 Aseed 家族的误差曲线与 base 重合。该方向关闭。

3.4 "纯 gauge 冗余是主因"（E2 的强形式）——已大幅降权

自 commit 31ab9a6 起 2gaussian_src/gaussian_terms.cpp:33-37 默认 \(A_{01}\)-only：COMdiag 两个 run 的 config 确认 tdvp_evolve_A_offdiag_only=1、param_dim=192（精确 gauge 已消除）。它们仍然失败，甚至更早。 caveat：该测试与 COMdiag basis / 宽窗混杂在一起，普通 basis 上的干净 E2 仍值得补一发短窗（见 R2），但期望值应当下调。

3.5 "缺 COM-diagonal reserve"（E4 的替换式实现）——反向证据

COMdiag basis 用 16 个双粒子 COM 对替换了 16 个单粒子 path pads。结果 gauss 的 0.01-越界点从 s≈0.131 提前到 s≈0.085–0.104，free 也全面变差 （free COMdiag 在 \(0.48\pi\) 处 0.0817 vs free base 0.0157），且 free 的宽度监控 \(\lambda_{\min}[\operatorname{Re}(A+B)]\) 早在 s≈0.15–0.21 就掉到 \(10^{-4}\)。 结论：单粒子 pads 是第一次跃迁的承重结构，不能被换掉。COM 对若有价值，正确形式是并集（加 K），而且要先由 R4 的探针证明确有 span 缺口。 静态投影 fidelity 高（0.9994）不代表 TDVP 动态可达——这是本轮实验最有教育意义的一课。

4. 仍然在场的失败机制（按优先级）

H1（P0）：未平衡 metric 上的全局截断冻结了跃迁所需的运动

\[ i\,C_{\alpha\beta}\dot z_\beta=g_\alpha, \qquad C_{\alpha\beta}=\langle\partial_\alpha\psi|\partial_\beta\psi\rangle, \qquad \partial_{B_k}\psi,\ \partial_{R_k}\psi,\ \partial_{A_k}\psi\ \propto\ u_k . \]
非-\(u\) 参数的切向量都带因子 \(u_k\)，metric 对角元 \(\propto|u_k|^2\)。pad 项 \(|u|\sim5.6\times10^{-4}\)–\(2.2\times10^{-3}\) ⇒ 其形状/位置自由度的奇异值比核心方向低 \(10^{6}\) 量级；在 \(\sigma_{\max}\sim10^{3}\)–\(10^{4}\) 下，任何 \(\mathrm{rcond}\ge10^{-6}\) 都把它们整段截掉。

低振幅项于是只剩线性的 \(u\) 通道可动。这个效应必须拆成两面看：对预置在泵浦路径上等待接收振幅的空 reserve，"被冻结"正是设计意图——它们本来就应该停在准备好的位置上，等波包到达时起作用（free + rcond 10⁻⁵ 正是以"pads 停车 + u 通道输运"的模式几乎通过 Gate 1 的，证明停车模式本身可行）。有害的是另外两件事：(a) 已占据项（core，以及 \(|u|\sim0.01\)–\(0.1\) 正在被填充的项）的 B/R/A 块之间天然尺度不同，在同一个全局阈值下部分该保留的方向被错误截断；(b) 正在苏醒的项骑在截断边界上，保留集合随步翻转（观测：跃迁窗内 rank 在 16–34 之间抖动），给投影动力学注入噪声。证据链： (i) 每个 gauss run 的 discarded_rhs_fraction 在 \(\phi\approx0.43\)–\(0.47\pi\) 冲到 0.21–0.56，与误差起点重合； (ii) N=1 成功 run 一生峰值只有 0.105；(iii) free 在 rcond 10⁻⁵（截得更少）后大幅改善。 处方：R2 去 gauge + R3 修订版"占据门控白化"——让 rcond 作用在已占据方向的真实共线性上，同时让沉睡的 reserve 保持停车（见 §6-R3 与 §8.2 的推导）。

H2（P0，与 H1 并列）：gauss 分支或许根本不需要演化 A——E1 从未测试

\[ V_g(x_0,x_1)=g\,e^{-(x_0-x_1)^2/\sigma^2},\qquad \frac{g}{E_R}=0.3,\qquad \max_\sigma\bigl|\Delta P_{\mathrm{gauss}}-\Delta P_{\mathrm{free}}\bigr|_{\mathrm{grid}}\approx1.3\times10^{-3}\text{–}5.8\times10^{-3}. \]
相互作用对输运的物理修正是 10⁻³ 量级；为了它把参数维数从 160 抬到 256/192、引入全部 A-块病态，很可能得不偿失（失败报告 §1 已提出此警告）。

free（A frozen，160 维）能通过第一次跃迁；把同一配置加上 Gaussian 相互作用哈密顿量（A 仍 frozen）——这个最显然的对照（失败报告矩阵中的 E1）至今没有跑过， 因为 2gaussian driver 从一开始就硬编码 evolve_A=true。若 E1 通过：直接得到最短成功路径（frozen-A gauss 全周期）；若失败：嫌疑立刻集中到 Gaussian 相互作用核/梯度，可用有限差分逐项排查。

H3（P1）：流形沿真实路径的表示能力仍未测量

精确的 free 两玻色子态是轨道积 \(|\varphi(t)\rangle\otimes|\varphi(t)\rangle\)；把 K₁=16 的单粒子和展开成对称化乘积需要 \(K_1(K_1+1)/2=136\) 个二维基函数，而 K=32 只覆盖对角积+pads—— 跃迁期轨道跨井离域时，需要的 cross-term 集合随时间移动，K32 能否动态地跟上从未被测量。失败报告的静态位移-fidelity 表只回答了"span 在不在"，没回答"TDVP 沿途够不够"。 处方：R4 表示能力探针——对 grid 波函数快照做变分拟合，把"流形不行"与"求解不行"一刀分开。

5. 两道关卡与成功标准

把"成功"拆成两道可独立验收的关卡，避免再用 1–6 天的全周期 run 去测试单一假设。

Gate 1 —— 通过第一次跃迁（短窗，s∈[0, 0.35]，几小时/发）

  • \(\max_{\phi\le0.52\pi}\bigl|\Delta P_{\mathrm{ECG}}-\Delta P_{\mathrm{grid}}\bigr|<0.01\)（失败报告的验收线，free rcond 10⁻⁵ 已经达到 0.006）。
  • discarded_rhs_fraction 峰值 < 0.10（以 N=1 成功 run 为标尺）。
  • \(\lambda_{\min}[\operatorname{Re}(A+B)]>10^{-2}\) 全程；能量沿 grid 瞬时值平滑变化、无台阶。
  • 对 rcond（10⁻⁴↔10⁻⁵）的轨迹敏感性显著小于对物理参数的敏感性。

Gate 2 —— 全周期稳定性（先 free 后 gauss）

  • free 全周期：\(\Delta P=-2\pm0.02\)；能量相对漂移 < 10⁻²；rank 不跌破 ~K/2；\(\lambda_{\min}[\operatorname{Re}(A+B)]\ge10^{-2}\)。
  • gauss σ=1 全周期：同标准，另加 \(\langle V_g\rangle\) 与 grid 的 \(r_{12}\) 曲线偏差 < 0.05。
  • 验收之后才恢复 8-σ sweep（R8）；在此之前任何 σ 扫描都是在为同一个错误付八倍机时。

为什么有信心：历史混合约定下，N=1 K16 已拿到 \(\Delta P=-0.9974\) 的干净单粒子泵浦；full-depth Hamiltonian + full-depth schedule 的现成 N=1 grid 基准为 \(\Delta x/a=-1.000759\)，而 N=2 grid 也确认目标态全程存在且量子化 −2。新协议仍需 ECG 回归，不能把旧 N=1 数字直接当作新 schedule 的预期值。

8.2 R3 门控白化（src/realtime_tdvp.cpp · compute_rhs_dz，~60–120 行）

先看朴素白化为什么会破坏 path-pad 策略（basis 设计意图：reserve 停在泵浦路径上，等被占据时才起作用）：

\[ \partial_\alpha\psi=u_k\,\chi_\alpha,\qquad C_{\alpha\alpha}=|u_k|^2\|\chi_\alpha\|^2,\qquad g_\alpha=u_k^{*}\,\langle\chi_\alpha|H|\psi\rangle \qquad(\alpha\in\{B,R,A\}\ \text{of term}\ k), \] \[ \text{朴素白化}\ \Longrightarrow\ \dot z_\alpha\ \sim\ \frac{\langle\hat\chi_\alpha|H|\psi\rangle}{|u_k|}, \qquad \hat\chi_\alpha=\chi_\alpha/\|\chi_\alpha\| . \]
速度带 \(1/|u_k|\) 放大因子：对 \(|u|\sim10^{-3}\)、停在活动区旁一两个井的 reserve，分子是 \(O(10^{-1})\) 的尾部重叠，速度被放大到 \(O(10^{2})\)——pad 在接收振幅之前就被拽离停车位或撕碎宽度。 这是 vMCG 文献中"弱占据 Gaussian 速度病态"的标准形态；它同时解释了为什么当前"按振幅加权 + 全局 rcond"意外地实现了正确的停车行为。远处 pads 分子指数小、只会缓慢漂移；受害最重的恰是泵浦区内即将被占据的 pads。

修订版两条实现路线（结论一致：只白化已占据流形，沉睡 reserve 必须继续停车）：

\[ \textbf{R3a 显式占据门：}\quad \alpha\in\{B,R,A\}\ \text{of term}\ k\ \text{参与演化} \iff |u_k|>u_{\mathrm{on}}\ (\text{建议 }0.02\text{–}0.05,\ \text{带迟滞}), \] \[ \textbf{R3b 振幅地板白化：}\quad D_\alpha=\max(|u_k|,\,u_{\mathrm{on}})^2\,\|\chi_\alpha\|^2,\qquad \tilde M=D^{-1/2}C\,D^{-1/2}+\tilde\lambda I,\qquad \tilde M\,\widetilde{\delta z}=D^{-1/2}(-i\,g),\qquad \dot z=D^{-1/2}\,\widetilde{\delta z}. \]
R3b 下沉睡 pad 的白化对角 \(=(|u_k|/u_{\mathrm{on}})^2\ll1\)，仍被 rcond 截断 ⇒ 继续停车；苏醒项速度上界 \(1/u_{\mathrm{on}}\)（有界的"类 spawning"预漂移，可通过调大 \(u_{\mathrm{on}}\) 关掉）。 u-块不做门控：\(\partial\psi/\partial u_k=\) Gaussian 本身，与 \(u\) 无关、天然 \(O(1)\)，而且它正是 pads 接收振幅的通道，必须常开。 \(\tilde\lambda\) 加在白化之后（相对正则化）；保留 raw-\(\lambda_C\) 变体做 A/B 对照。SVD 截断照旧 \(\sigma_i<\mathrm{rcond}\cdot\tilde\sigma_{\max}\)。 历史注：归档的 N=1 二进制曾有 c_whiten 开关与 relative_whitened_residual 列，现树中已无此代码——需要重新实现，但有先例可循。
  • 停车验收（硬性）：从 snapshots.csv（每 250 步存全量基参数）离线检查：所有 \(|u_k|<u_{\mathrm{on}}\) 的 reserve 满足 \(\|R_k(t)-R_k(0)\|<1\)（代码单位，约 1/4 个短晶格周期）且 \(\lambda_{\min}[\operatorname{Re}(A_k+B_k)]\) 不漂。验收失败 ⇒ 调大 \(u_{\mathrm{on}}\) 或退回 R3a。
