# Vs3/Vl3 full-depth schedule convention

*Current convention adopted 2026-07-10. Historical mismatch recorded below.*

## Canonical protocol

All new Vs3/Vl3 ECG and grid runs use the same full-depth lattice,

\[
V(x,\phi)
=
-3E_R\cos(4\pi x/a)
-3E_R\cos(2\pi x/a+\phi),
\qquad a=8,
\]

both to compute the instantaneous first-band gap and to evolve the wavefunction.
The canonical phase schedule is:

- `gap_adaptive_vs3vl3_full_depth_schedule.csv`
- generated by `make_vs3vl3_full_depth_schedule.py`
- rule: \(d\phi/dt\propto g(\phi)^2\), equivalently
  \(ds/d\phi\propto 1/g(\phi)^2\)
- verified minimum gap:
  \(g_{\min}=0.146121701622\) in code units, or
  \(g_{\min}/E_R=0.473767160454\)

Regenerate it from the repository root with:

```bash
python3 rice_mele_reference/Vs3Vl3_3_3/make_vs3vl3_full_depth_schedule.py
```

The C++ configuration records
`phase_schedule_convention=full_depth_Vs3_Vl3`, and new output leaf directories
carry the suffix `_full_depth_schedule` so they cannot be confused with the old
protocol.

## Historical N=1 success case

The archived N=1, K16 success result
\(\Delta P=-0.9973555596\) did **not** use the new consistent protocol. It used:

| Component | Historical setting |
|---|---|
| evolved Hamiltonian | full depth, cosine coefficients \(-3E_R\) |
| schedule gap | half depth, cosine amplitudes \(-1.5E_R\) |
| schedule file | `gap_adaptive_vs3vl3_maincpp_schedule.csv` |
| schedule minimum gap | \(g_{\min}/E_R=0.562416987053\) |

That old CSV is retained unchanged for reproducibility, but it is no longer the
default for any current Vs3/Vl3 driver. Its `maincpp` name is historical and must
not be interpreted as the current C++ Hamiltonian convention.

## Why the mismatch mattered

The old schedule was produced with a factor \(1/2\) in both cosine amplitudes,

\[
V_{\mathrm{old\ schedule}}(x,\phi)
=
-\frac{3}{2}E_R\cos(4\pi x/a)
-\frac{3}{2}E_R\cos(2\pi x/a+\phi),
\]

while ECG and N=2 grid evolution used the full-depth potential above. ECG and
grid therefore remained comparable to each other, but the adaptive rate was not
optimized for the Hamiltonian they evolved. The shallower lattice has a different
gap profile and the larger minimum gap quoted above.

## Data provenance

The following files remain valid without rerunning:

- `grid_rm_curves_Vs3Vl3.npz`: full-depth static band/Rice–Mele curves.
- `gap_adaptive_grid_Vs3Vl3.npz`: full-depth N=1 grid evolution driven by a
  full-depth-derived schedule.

The following are historical or use the retired convention:

- `gap_adaptive_vs3vl3_maincpp_schedule.csv`: half-depth-derived schedule.
- `gap_adaptive_grid_Vs3Vl3_maincpp.npz`: half-depth Hamiltonian and schedule.
- `gapadaptive_grid_ref_T160pi.npz`: full-depth Hamiltonian driven by the old
  half-depth-derived schedule.
- existing `out/**` Vs3/Vl3 ECG and N=2 grid results created before the
  2026-07-10 switch: full-depth evolution with the old schedule.

Those historical results must not be relabelled as full-depth-schedule results.
Any comparison under the new protocol requires rerunning both ECG and its matched
grid reference with `gap_adaptive_vs3vl3_full_depth_schedule.csv`.
